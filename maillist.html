Maillist Redesign

Maillist is an internal web application created by Simon Fraser University's developers for the creation and administration of email mailing lists.

I was asked to produce some conceptual redesigns of the application, aimed at improving usability, aesthetics and making it more self-service.

Process

Usability Analysis

I identified three main pain points in the Maillist creation process:

  • Poor feedback: The interface did not let the user know where they were in the process of creating a mailing list, and did not let the user know if the creation of a list was successful after completion.
  • A lack of interface consistency: There are a number of different forms for creating different types of mail lists, but all the forms looked slightly different. The interface also did not reflect SFU's web branding and style, which led to confusion on the part of users.
  • Poor form field labelling and on-screen assistive text: Form fields were rife with insider language, and onscreen instructions were unclear.

Design Process

I proceeded with a redesign based on the following three recommendations:

  • The interface should provide feedback: for multi-screen forms, the user should be visually prompted as to where they are, and should receive a confirmation message upon interface completion.
  • The interface should reflect SFU's web branding and style, and there should be visual consistency in things like form field size and labellng between different mail list creation forms.
  • The interface should not assume the user knows anything: labels and form options should be explained in plain english, and each screen should provide a full explanation of its function, and where users can go for extra support if required.
